Apple Notes Privacy Export Format

This document describes the format produced by Apple's EU privacy data export (privacy.apple.com), specifically for Notes. It was reverse-engineered from a real export in April 2026. The brains import pipeline is built against this specification.

Key insight: The format is plain UTF-8 text, not HTML or XML. Structure is encoded using Unicode characters and tab indentation, not markup tags.

1. Zip Structure

Apple delivers the export as a zip file. Inside it:

[archive-name].zip
└── iCloud Notes/
    ├── Notes/                      # all live notes
    ├── Recently Deleted/           # deleted notes (same structure)
    ├── Notes Details.csv           # metadata: dates, pinned, deleted
    ├── Shared Notes Info.csv       # notes you shared with others
    └── Subscribed Notes Info.csv   # notes others shared with you

2. Notes Directory Structure

Each note is stored as a .txt file inside a wrapper directory. The directory name equals the note title. The structure encodes Apple Notes folder membership through directory depth: 

Notes/
├── [NoteTitle]/                      # root-level note (no folder)
│   └── [NoteTitle].txt
│
├── [NoteTitle]/                      # second note with same title
│   ├── [NoteTitle].txt               # → [NoteTitle]-1.txt, -2.txt etc.
│   └── [NoteTitle]-1.txt
│
├── [AppleFolder]/                    # an Apple Notes folder
│   ├── [NoteTitle]/
│   │   └── [NoteTitle].txt           # note inside the folder
│   └── [AnotherNote]/
│       ├── [AnotherNote].txt
│       └── photo.jpg                 # attachment in same directory

Depth rules

Path depth from Notes/Interpretation
Notes/[Wrap]/[Wrap].txt (depth 2)Root-level note, title = [Wrap]
Notes/[Folder]/[Wrap]/[Wrap].txt (depth 3)Note in folder [Folder], title = [Wrap]

Duplicate title suffix

When multiple notes share the same title in a folder, Apple appends -N to the filename (not the wrapper directory). Strip this suffix to get the canonical title: 

✅/✅.txt    → title: "✅"
✅/✅-1.txt  → title: "✅"   # separate note, same title
✅/✅-2.txt  → title: "✅"   # yet another

Attachments

Images, PDFs and audio files are stored as siblings of the .txt file in the same wrapper directory. They are not referenced by filename inside the text — their presence is indicated by ￼ (Object Replacement Character) inline in the note body. 

Houses to have/
├── Houses to have.txt    # body contains several ￼ placeholders
├── IMG_4846.JPG
├── IMG_7512.jpeg
└── IMG_7514.jpeg

3. Note File Format

Each .txt file is UTF-8, Unix line endings (\n). The first line is the note title. A blank line follows, then the body. 

Packliste                          # ← title (line 1)
                                    # ← blank separator
	◦	Zip-Jacken               # ← unchecked checklist item
	◦	Sonnenbrille
	✓	Shorts                   # ← checked checklist item
	✓	Jacke

Einkaufen

- Brot                             # ← dash bullet
- Milch
* Käse                             # ← asterisk bullet (same meaning)

Some plain paragraph text.

4. Formatting Characters

All formatting is expressed through specific Unicode characters and tab characters. There are no HTML tags, Markdown symbols, or proprietary binary sequences.

Visual appearance Unicode Bytes (hex) Pattern in file Meaning
◦ U+25E6 09 e2 97 a6 09 TAB + ◦ + TAB + text Unchecked checklist item
✓ U+2713 09 e2 9c 93 09 TAB + ✓ + TAB + text Checked checklist item
- ASCII hyphen 2d 20 - text Bullet list item
* ASCII asterisk 2a 20 * text Bullet list item (alternate)
￼ U+FFFC ef bf bc inline in text Embedded image / attachment placeholder
(tab) U+0009 09 leading tabs Indentation / nesting depth

What Apple Notes styles look like in the export

Important: Apple Notes has styled text (Title, Heading, Subheading, Body, Monospaced). None of these are preserved in the privacy export. All styled text is exported as plain text with no markers. There is no way to distinguish a "Heading" from a "Body" line in the export format.

7. Subscribed Notes Info.csv

File Name, Owner Details, Last Modified Date, Shared On, Participants Details

Owner format: Name:X; Email:Y (email partially masked).

9. Edge Cases & Parser Notes

  • Title mismatch: The first line of the .txt file should match the wrapper directory name, but always prefer the file content as the canonical title.
  • Empty notes: A note may contain only the title line and nothing else, or only U+FFFC placeholders after the title.
  • Mixed lists: Checked and unchecked checklist items can be interleaved in the same block. They should all go into one taskList TipTap node.
  • Checklist followed by plain text: A note can have a checklist section, then a blank line, then paragraphs. These are separate blocks.
  • Duplicate sections: Some notes appear to repeat their content (e.g. Packliste has the checklist and then a plain-text copy of the same items below a second "Packliste" heading). This is a quirk of Apple's export, not an error — both sections should be imported as-is.
  • Emoji and special characters in folder/note names: Fully supported. Folder names like ✅, 🤔, Даниел, •, " are all valid.
  • URL encoding: Some URLs observed with | in place of /. This appears to be a display artefact from copy-pasting, not an Apple encoding — treat as plain text.
  • Recently Deleted: Skip by default. The same notes appear in Notes Details.csv with Deleted: Yes.
