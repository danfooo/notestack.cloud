Apple Notes Privacy Export Format

This document describes the format produced by Apple's EU privacy data export (privacy.apple.com), specifically for Notes. It was reverse-engineered from a real export in April 2026. The brains import pipeline is built against this specification.

Key insight: The format is plain UTF-8 text, not HTML or XML. Structure is encoded using Unicode characters and tab indentation, not markup tags.

1. Zip Structure

Apple delivers the export as a zip file. Inside it:

[archive-name].zip
└── iCloud Notes/
    ├── Notes/                      # all live notes
    ├── Recently Deleted/           # deleted notes — same structure, imported with deleted_at set
    ├── Notes Details.csv           # metadata: dates, pinned, deleted
    ├── Shared Notes Info.csv       # notes you shared with others
    └── Subscribed Notes Info.csv   # notes others shared with you

2. Notes Directory Structure

Each note is stored as a .txt file inside a wrapper directory. The wrapper directory name is always the note title — it is the canonical source for the title, not the filename. The structure encodes Apple Notes folder membership through directory depth: 

Notes/
├── Meeting notes/                    # root-level note (no Apple folder)
│   └── Meeting notes.txt
│
├── Project Alpha/                    # root-level note with same title as another
│   ├── Project Alpha.txt             # → Apple appends -N to the FILE only
│   └── Project Alpha-1.txt          # both have title "Project Alpha"
│
├── Work/                             # an Apple Notes folder named "Work"
│   ├── Q3 planning/
│   │   └── Q3 planning.txt           # note inside "Work"
│   └── Team updates/
│       ├── Team updates.txt
│       └── Attachment.png            # image attachment (see §5)

Depth rules

Path depth from Notes/Interpretation
Notes/[Wrap]/[Wrap].txt (depth 2)Root-level note. Title = [Wrap] (the directory name).
Notes/[Folder]/[Wrap]/[Wrap].txt (depth 3)Note in Apple folder [Folder]. Title = [Wrap].

Duplicate title suffix

When multiple notes share the same title in the same folder, Apple appends -N to the filename only, never to the wrapper directory. The directory name is always the original, unmodified title. 

Project Alpha/Project Alpha.txt   → title: "Project Alpha"
Project Alpha/Project Alpha-1.txt → title: "Project Alpha"  # different note, same title
Project Alpha/Project Alpha-2.txt → title: "Project Alpha"  # yet another
Implication: Never derive the title from the filename. A note genuinely titled "Sprint-3" and an Apple-disambiguated duplicate called "Sprint-3.txt" are indistinguishable by filename alone. Always use the wrapper directory name.

Folder names with special characters

Note titles (and therefore directory names) can contain any Unicode character: emoji, Cyrillic, Arabic, CJK, punctuation. macOS does not allow / in directory names, so Apple replaces / with | in the filesystem path when a note's title contains a forward slash (common when people title notes after URLs or file paths). The content inside the .txt file is unaffected — URLs in the body use normal /. 

# Note titled: "https://example.com/api/docs"
# Becomes directory: "https:||example.com|api|docs/"
# Inside the .txt file the URL reads: https://example.com/api/docs

Each .txt file is UTF-8, Unix line endings (\n). The first line is the note title (same text as the wrapper directory name). A blank line follows, then the body.

4. Formatting Characters

All formatting is expressed through specific Unicode characters and tab characters. There are no HTML tags, Markdown symbols, or proprietary binary sequences.

✓ U+2713 09 e2 9c 93 09 TAB + ✓ + TAB + text Checked checklist item
- ASCII hyphen 2d 20 - text Dash-style bullet list item

￼ U+FFFC ef bf bc inline in text Embedded image / attachment placeholder (see §5)

Bullet list style

The two markers are treated as distinct list styles in brains:

  • * text → standard bulletList (round bullets, the TipTap default)
  • - text → bulletList with attrs.listStyle = "dash" (rendered as dash bullets via a client TipTap extension)

A run of consecutive - items forms one dash list; a run of consecutive * items forms one bullet list. Switching markers mid-sequence starts a new list.

What Apple Notes styles are lost in the export

Important: Apple Notes has rich text styles (Title, Heading, Subheading, Body, Monospaced, Bold, Italic, Underline). None of these are preserved in the privacy export. All styled text is exported as plain text with no markers. There is no way to distinguish a "Heading" line from a "Body" line in the export.

5. Attachments and Inline Images

Images, PDFs, and audio files are stored as siblings of the .txt file in the same wrapper directory. Their position within the note is marked by U+FFFC (Object Replacement Character, ￼) inline in the text.

Naming convention and ordering

Apple names attachments using the same suffix convention as note files. The first attachment is Attachment.png (no suffix); subsequent ones are Attachment-1.png, Attachment-2.png, and so on. The order of U+FFFC occurrences in the text (top to bottom) maps directly to this sorted attachment list: 

Team updates/
├── Team updates.txt        # body: "Header text\n￼\nMiddle text\n￼\nFooter"
├── Attachment.png          # → replaces first  ￼
├── Attachment-1.png        # → replaces second ￼
└── Attachment-2.png        # → replaces third  ￼ (if present)

Sort order: no-suffix file first, then -1, -2, etc. Do not sort alphabetically — Attachment-10.png sorts after Attachment-9.png, not after Attachment-1.png.

Notes containing only images

A note may contain nothing but U+FFFC characters after the title — meaning the user's entire content was photos or scanned documents with no typed text. The note body in this case is a sequence of one U+FFFC per image, each on its own line. Imported into brains, these become a sequence of inline image nodes with no surrounding text.

7. Shared Notes Info.csv

File Name, Shared On, Last Modified Date, Participants Details

Participants are encoded as a pipe-separated list of semicolon-delimited fields:

Work/Q3 planning/Q3 planning.txt,01-15-2024 10:00:00,01-20-2024 15:30:00,"Name:Alice Müller; Email:a*****@example.de; Permission:READ_WRITE; Acceptance Status:ACCEPTED | Name:Yuki Tanaka; Email:y*****@example.jp; Permission:READ_ONLY; Acceptance Status:INVITED"

Emails are always partially masked by Apple (e.g. a*****@example.de). Display names are unmasked.

8. Subscribed Notes Info.csv

File Name, Owner Details, Last Modified Date, Shared On, Participants Details

Owner format: Name:X; Email:Y (same masking). Example:

10. Recently Deleted

The Recently Deleted/ directory has the same structure as Notes/ and contains notes the user deleted before their export was generated. These notes also appear in Notes Details.csv with Deleted: Yes.

brains imports Recently Deleted notes with deleted_at set to the time of import. They are immediately in the soft-deleted state and will be permanently removed by the 30-day deferred hard-deletion cron job unless the user restores them first.

30-day grace period: All soft-deleted notes (whether deleted by the user in brains or imported from Recently Deleted) are permanently hard-deleted 30 days after deleted_at. A daily cron job enforces this.
